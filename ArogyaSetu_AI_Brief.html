🔴 The Problem

India has 10,088 healthcare facilities listed online — but the listings lie.

847 hospitals claim 24/7 emergency care but have fewer than 3 doctors.
168 hospitals claim ICU capability with less than 2 doctors.
2,884 facilities have specialty data truncated at exactly 50 — a scraping bug hiding their real capacity.

During COVID-19, my family in a small town in Gujarat, India searched for a hospital to treat pneumonia. They found one listed online. When they arrived — there was no doctor. They had to travel hours to find actual care.

For 1.4 billion people, trusting a hospital listing can be a life-or-death decision. Current platforms show you a name, an address, and a list of claimed services. None of them tell you: should you actually trust this?

🟢 How We Solved It

We built ArogyaSetu AI — a Facility Trust Desk that doesn't just show data. It cross-examines it.

🤥 Pinocchio Score™ — Overclaiming Detection

Every facility gets a score (0–100) measuring overclaiming risk using 8 penalty factors:

  • "You claim 24/7 emergency, but you have fewer than 3 doctors" → +15
  • "You claim ICU, but you have fewer than 2 doctors" → +25
  • "You claim multispecialty, but fewer than 3 doctors" → +20
  • "You claim cardiology, but no cardiac equipment listed" → +10

Every penalty is traceable to a specific data field. Judges can click any score and see exactly why.

🔍 Evidence Triangulation — Text Citation Engine

For every claim (ICU, 24/7, Cardiology, NICU), we scan 4 raw text fields: description, capability, procedure, and equipment.

  • Found? → ✅ description: "...provides 24/7 emergency services..." — exact keyword highlighted
  • Not found? → ⊘ equipment: not mentioned

Each claim gets a verdict: Strong, Partial, or Weak evidence — all traceable to source text.

📊 Uncertainty Communication

Only 36.4% of facilities report doctor count. Only 25.1% report bed capacity. When data is missing, we say so explicitly — field by field, with dataset-wide percentages. We never present weak evidence as fact.

🎯 AI Policy Advisor

An autonomous agent powered by Llama 3.3 70B (via Databricks Foundation Model API) that:

  • Queries live facility data from Unity Catalog via SQL
  • Cross-references claims against structural evidence
  • Generates actionable intelligence briefs with citations
  • Supports voice input in 6 Indian languages (Hindi, Tamil, Bengali, Telugu, Marathi, Gujarati)

⭐ What Makes Us Unique

1. We show evidence, not just scores

Other dashboards show a number and say "trust this." We show the raw text citation, the penalty breakdown, and the data gaps — so you can verify every claim yourself.

2. We keep suspicious hospitals visible

We don't filter bad hospitals out. We explain WHY they can't be trusted. A health planner needs to know a hospital exists but shouldn't receive referrals — not that it disappeared from the system.

3. We say "we don't know"

Almost no healthcare platform communicates uncertainty. We do, field by field. If doctor count is missing, we show an amber warning — not a fake confidence score.

4. 100% Databricks Stack

Zero external dependencies. Data never leaves the platform:

  • Unity Catalog — Data governance
  • SQL Warehouse — Serverless compute
  • Foundation Model API — Llama 3.3 70B
  • Agent Bricks + MLflow — Agent observability & tracing
  • Databricks Apps — Live production deployment

5. This is personal

This isn't a hypothetical use case built for a hackathon. A family member couldn't find a real hospital during COVID-19. We built what should have existed.
